## 1. O que é uma estrutura condicional em programação? Dê um exemplo do cotidiano usando "se... então... senão".
Uma estrutura condicional é usada para tomar decisões em um programa. Dependendo de uma condição, uma ação será executada.
Exemplo: Se eu tiver dinheiro, então compro um lanche; senão, levo comida de casa.

## 2. Qual é a diferença entre usar apenas if e usar if/else?
O if executa uma ação somente quando a condição é verdadeira.
O if/else executa uma ação quando a condição é verdadeira e outra quando é falsa.

## 3. Quando usamos else if? Crie um exemplo com três ou mais possibilidades.
Usamos else if quando existem várias possibilidades de resultado.
Em uma escola:
Se a nota for 9 ou mais, o aluno recebe Excelente.
Senão, se a nota for 7 ou mais, recebe Bom.
Senão, se a nota for 5 ou mais, recebe Regular.

## 4. O que significa uma condição retornar true ou false?
Significa que a condição pode ser verdadeira (true) ou falsa (false).

## 5. Qual é a diferença entre == e === em JavaScript? Qual deles é mais recomendado?
O operador == compara apenas os valores.
O operador === compara os valores e os tipos.
O mais recomendado é o === porque evita erros de comparação.

## 6. Para que servem os operadores &&, || e !? Explique com um exemplo para cada um.
&& (E): todas as condições precisam ser verdadeiras.
idade >= 18 && possuiCNH

|| (OU): pelo menos uma condição precisa ser verdadeira.
temDinheiro || temCartao

! (NÃO): inverte o valor da condição.
!estaChovendo

## 7. O que o operador % faz? Por que ele é útil para descobrir se um número é par ou ímpar?
O operador % retorna o resto da divisão.
Ele é útil porque um número par dividido por 2 sempre tem resto 0.

## 8. O que é switch case? Em que situação ele fica melhor do que vários else if?
Switch case é uma estrutura usada para comparar uma variável com vários valores possíveis.
Ele fica melhor quando há muitas opções fixas.

## 9. Para que servem break e default dentro de um switch?
O break encerra o caso atual e impede que os próximos sejam executados.
O default é executado quando nenhum dos casos corresponde ao valor informado.

## 10. Por que é importante testar o mesmo programa com vários valores diferentes?
Porque ajuda a encontrar erros e garantir que o programa funcione corretamente em diferentes situações.
